新機能 "$ARGUMENTS" の仕様書を作成する。

## 参照
- docs/要件定義書.md（対応する FR-ID を特定）
- specs/_TEMPLATE.md（この形式に従う）
- CLAUDE.md（ドメインモデル・規約）

## 手順
1. 要件定義書から対応する FR を特定し、目的と優先度を確認
2. specs/ 内の既存ファイルを確認して連番（NNN）を決める
3. specs/NNN-<feature>.md を作成
4. テンプレートの全項目を埋める：ユーザーフロー / 画面要素 / 状態 / エッジケース / 受け入れ条件 / スコープ外
5. 受け入れ条件は検証可能な箇条書きにする（実装と PR はこれに照らして判定）
6. 不明点は仮置きせず「未決事項」に明示し、確認を求める
